## Overview

The `ChangesTable` component is a feature-rich table component that displays database changes with dynamic column generation, real-time filtering, multi-column sorting, and operation-based color coding. It adapts to any schema structure by dynamically generating columns from the data.

## Dynamic Column Generation

The component automatically detects all unique columns from the changes array:

```typescript
const columns = useMemo(() => {
  const columnSet = new Set<string>();
  changes.forEach((change) => {
    Object.keys(change).forEach((key) => columnSet.add(key));
  });
  return Array.from(columnSet);
}, [changes]);
```

This approach ensures the table adapts to any database schema without hardcoded column definitions.

## Filtering Logic

The component implements case-insensitive, multi-column filtering:

```typescript
const filteredChanges = useMemo(() => {
  const filterEntries = Object.entries(filters).filter(([_, value]) => value.trim() !== "");

  if (filterEntries.length === 0) {
    return changes;
  }

  return changes.filter((change) => {
    return filterEntries.every(([column, filterValue]) => {
      const value = change[column];
      if (value == null) {
        return filterValue.toLowerCase() === "null" || filterValue.toLowerCase() === "";
      }
      return String(value).toLowerCase().includes(filterValue.toLowerCase());
    });
  });
}, [changes, filters]);
```

### Filter Features

- **Case-insensitive**: All comparisons are lowercase
- **Substring matching**: Uses `.includes()` for partial matches
- **Multi-column**: All active filters must match (AND logic)
- **Null handling**: `null` values can be filtered by typing "null"
- **Real-time**: Updates immediately as user types

## Sorting Logic

The component implements intelligent multi-type sorting:

```typescript
const sortedChanges = useMemo(() => {
  if (!sortColumn || !sortDirection) {
    return filteredChanges;
  }

  return [...filteredChanges].sort((a, b) => {
    const aValue = a[sortColumn];
    const bValue = b[sortColumn];

    // Handle null/undefined values
    if (aValue == null && bValue == null) return 0;
    if (aValue == null) return 1;
    if (bValue == null) return -1;

    // Compare values
    let comparison = 0;
    if (typeof aValue === "number" && typeof bValue === "number") {
      comparison = aValue - bValue;
    } else if (aValue instanceof Date && bValue instanceof Date) {
      comparison = aValue.getTime() - bValue.getTime();
    } else {
      // Try to parse as date (ISO format)
      const aDate = typeof aValue === "string" ? new Date(aValue) : null;
      const bDate = typeof bValue === "string" ? new Date(bValue) : null;

      if (aDate && bDate && !isNaN(aDate.getTime()) && !isNaN(bDate.getTime())) {
        comparison = aDate.getTime() - bDate.getTime();
      } else {
        // String comparison
        comparison = String(aValue).localeCompare(String(bValue));
      }
    }

    return sortDirection === "asc" ? comparison : -comparison;
  });
}, [filteredChanges, sortColumn, sortDirection]);
```

### Sort Features

- **Type-aware**: Handles numbers, dates, and strings differently
- **Null handling**: Null values always sort to the bottom
- **Date detection**: Automatically parses ISO date strings
- **Three-state cycle**: Click header to cycle through ascending → descending → no sort
- **Visual indicators**: Shows ↑ (asc), ↓ (desc), or ↕ (no sort) arrows

### Sort Interaction

```typescript
const handleSort = (column: string) => {
  if (sortColumn === column) {
    // If already sorted by this column, change direction
    if (sortDirection === "asc") {
      setSortDirection("desc");
    } else if (sortDirection === "desc") {
      setSortColumn(null);
      setSortDirection(null);
    }
  } else {
    // New column, sort ascending
    setSortColumn(column);
    setSortDirection("asc");
  }
};
```

Click sequence: **unsorted** → **ascending** → **descending** → **unsorted**

## Operation Color Coding

The component uses color-coded badges for operation types:

```typescript
const getOperationColor = (operation: string) => {
  switch (operation.toUpperCase()) {
    case "INSERT":

    case "UPDATE":
      return "#3b82f6"; // blue
    case "DELETE":
      return "#ef4444"; // red
    default:
      return "#6b7280"; // gray
  }
};
```

## Performance Optimizations

- **useMemo hooks**: Column generation, filtering, and sorting are memoized to prevent unnecessary recalculations
- **Dependency arrays**: Each memoized value only recomputes when relevant dependencies change
- **Immutable sorting**: Creates new array with spread operator to avoid mutating filtered results

## Accessibility Features

- **Keyboard navigation**: All interactive elements are keyboard accessible
- **Visual feedback**: Hover states on sortable headers and rows
- **Clear indicators**: Sort arrows and filter count provide clear state feedback
- **Placeholder text**: Input placeholders indicate filtering functionality
- **Null value display**: Clearly marks null values with italic "null" text
