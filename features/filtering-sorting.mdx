## Overview

The ChangesTable component provides powerful client-side filtering and sorting capabilities. Users can filter data by any column and sort by multiple columns with direction toggles.

## Per-Column Text Filters

Each column has its own filter input that performs case-insensitive substring matching:

```typescript src/components/ChangesTable.tsx
const [filters, setFilters] = useState<Record<string, string>>({});

// Function to update a filter
const handleFilterChange = (column: string, value: string) => {
  setFilters((prev) => ({
    ...prev,
    [column]: value,
  }));
};
```

## Filter Matching Logic

Filters use case-insensitive substring matching with null value handling:

```typescript src/components/ChangesTable.tsx
// Function to filter changes
const filteredChanges = useMemo(() => {
  const filterEntries = Object.entries(filters).filter(([_, value]) => value.trim() !== "");

  if (filterEntries.length === 0) {
    return changes;
  }

  return changes.filter((change) => {
    return filterEntries.every(([column, filterValue]) => {
      const value = change[column];
      if (value == null) {
        return filterValue.toLowerCase() === "null" || filterValue.toLowerCase() === "";
      }
      return String(value).toLowerCase().includes(filterValue.toLowerCase());
    });
  });
}, [changes, filters]);
```

<Info>
  All filters must match for a row to be displayed (AND logic). Type "null" to filter for null values.
</Info>

## Multi-Column Sorting

Click any column header to sort. Click again to reverse direction, and a third time to clear sorting.

### Sort State Management

```typescript src/components/ChangesTable.tsx
const [sortColumn, setSortColumn] = useState<string | null>(null);
const [sortDirection, setSortDirection] = useState<SortDirection>(null);

type SortDirection = "asc" | "desc" | null;
```

### Sort Toggle Logic

```typescript src/components/ChangesTable.tsx
// Function to handle header click
const handleSort = (column: string) => {
  if (sortColumn === column) {
    // If already sorted by this column, change direction
    if (sortDirection === "asc") {
      setSortDirection("desc");
    } else if (sortDirection === "desc") {
      setSortColumn(null);
      setSortDirection(null);
    }
  } else {
    // New column, sort ascending
    setSortColumn(column);
    setSortDirection("asc");
  }
};
```

<Steps>
  <Step title="First Click">
    Sort ascending (A→Z, 0→9, oldest→newest)
  </Step>
  <Step title="Second Click">
    Sort descending (Z→A, 9→0, newest→oldest)
  </Step>
  <Step title="Third Click">
    Clear sorting, return to original order
  </Step>
</Steps>

## Sort Comparisons

The sort function intelligently handles different data types:

```typescript src/components/ChangesTable.tsx
// Function to sort changes (after filtering)
const sortedChanges = useMemo(() => {
  if (!sortColumn || !sortDirection) {
    return filteredChanges;
  }

  return [...filteredChanges].sort((a, b) => {
    const aValue = a[sortColumn];
    const bValue = b[sortColumn];

    // Handle null/undefined values
    if (aValue == null && bValue == null) return 0;
    if (aValue == null) return 1;
    if (bValue == null) return -1;

    // Compare values
    let comparison = 0;
    if (typeof aValue === "number" && typeof bValue === "number") {
      comparison = aValue - bValue;
    } else if (aValue instanceof Date && bValue instanceof Date) {
      comparison = aValue.getTime() - bValue.getTime();
    } else {
      // Try to parse as date (ISO format)
      const aDate = typeof aValue === "string" ? new Date(aValue) : null;
      const bDate = typeof bValue === "string" ? new Date(bValue) : null;

      if (aDate && bDate && !isNaN(aDate.getTime()) && !isNaN(bDate.getTime())) {
        comparison = aDate.getTime() - bDate.getTime();
      } else {
        // String comparison
        comparison = String(aValue).localeCompare(String(bValue));
      }
    }

    return sortDirection === "asc" ? comparison : -comparison;
  });
}, [filteredChanges, sortColumn, sortDirection]);
```

## Performance Optimization

Both filtering and sorting use `useMemo` to avoid unnecessary recalculations:

```typescript src/components/ChangesTable.tsx
// Filters recalculate only when changes or filters change
const filteredChanges = useMemo(() => {
  // ... filter logic
}, [changes, filters]);

// Sorts recalculate only when filtered data or sort state changes
const sortedChanges = useMemo(() => {
  // ... sort logic
}, [filteredChanges, sortColumn, sortDirection]);
```

<Note>
  Filtering happens first, then sorting is applied to the filtered results. This ensures optimal performance and expected behavior.
</Note>
