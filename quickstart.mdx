## Prerequisites

Before you begin, ensure you have:

- **Bun** installed ([installation guide](https://bun.sh/docs/installation))
- **Docker** installed ([get Docker](https://docs.docker.com/get-docker/))
- **Git** (to clone the repository)

<Note>
  This guide assumes you're running on Linux, macOS, or WSL2 on Windows.
</Note>

## Getting Started

Follow these steps to get the monitor running:

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine:

    ```bash
    git clone https://github.com/miguelmartinezmestre/listen-postgres-changes-viewer.git
    cd listen-postgres-changes-viewer
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies using Bun:

    ```bash
    bun install
    ```

    This will install:
    - `postgres` (v3.4.7) - PostgreSQL client with logical replication support
    - `react` (v19.2.0) - React framework
    - `react-dom` (v19.2.0) - React DOM renderer
    - `typescript` (v5.9.3) - TypeScript compiler

    <Tip>
      Bun's package installation is typically 10-100x faster than npm!
    </Tip>
  </Step>

  <Step title="Start PostgreSQL">
    Start the PostgreSQL database using Docker Compose:

    ```bash
    docker-compose up -d
    ```

    This starts a PostgreSQL 16 container with:
    - Logical replication enabled (`wal_level=logical`)
    - Port 5432 exposed
    - Database: `appdb`
    - User: `appuser`
    - Password: `secret`
    - Publication: `alltables` (for all tables)

    Verify the database is running:

    ```bash
    docker ps | grep postgres-realtime
    ```
  </Step>

  <Step title="Start the application">
    Start the Bun server with Hot Module Reloading:

    ```bash
    bun run dev
    ```

    You should see:

    ```
    🚀 Server running at http://localhost:3000
    📡 WebSocket available at ws://localhost:3000/ws
    ✅ Realtime subscription ready (connected or reconnected)
    ```
  </Step>

  <Step title="Open in browser">
    Navigate to [http://localhost:3000](http://localhost:3000) in your web browser.

    You should see:
    - The application title: "📊 PostgreSQL Realtime Monitor"
    - A green "Connected" indicator
    - A message: "Waiting for database changes..."
  </Step>
</Steps>

## Test with Sample Data

Now let's make some database changes to see the real-time monitoring in action!

<Steps>
  <Step title="Connect to PostgreSQL">
    Open a new terminal and connect to the database:

    ```bash
    docker exec -it postgres-realtime psql -U appuser -d appdb
    ```
  </Step>

  <Step title="Create a table">
    Create a sample `todos` table:

    ```sql
    CREATE TABLE todos (
      id SERIAL PRIMARY KEY,
      title TEXT NOT NULL,
      done BOOLEAN NOT NULL DEFAULT false,
      created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    );
    ```

    <Note>
      The table is automatically included in the `alltables` publication, so changes will be streamed immediately.
    </Note>
  </Step>

  <Step title="Insert data">
    Insert some test records:

    ```sql
    INSERT INTO todos (title) VALUES ('Buy groceries');
    INSERT INTO todos (title) VALUES ('Write documentation');
    INSERT INTO todos (title) VALUES ('Deploy application');
    ```

    Check your browser - you should see three new rows appear instantly with:
    - Green **INSERT** badges
    - Table name: `public.todos`
    - All column values (id, title, done, created_at)
  </Step>

  <Step title="Update data">
    Mark a todo as complete:

    ```sql
    UPDATE todos SET done = true WHERE id = 1;
    ```

    You'll see a new row with:
    - Blue **UPDATE** badge
    - The updated values
  </Step>

  <Step title="Delete data">
    Remove a todo:

    ```sql
    DELETE FROM todos WHERE id = 2;
    ```

    A new row appears with:
    - Red **DELETE** badge
    - The values of the deleted record
  </Step>
</Steps>

## What to Expect

Once everything is running, here's what you'll see:

### Connection Status

At the top of the page:
- **Green indicator**: Connected to WebSocket server
- **Red indicator**: Disconnected (automatic reconnection after 3 seconds)
- **Total changes counter**: Shows the cumulative number of changes

### Real-time Table Interface

The main table displays:

1. **Column Headers**: Dynamically generated from all change data
   - Click any header to sort (ascending → descending → unsorted)
   - Shows sort direction with ↑ (ascending) or ↓ (descending) arrows

2. **Filter Row**: Below headers
   - Filter any column with case-insensitive search
   - Active filters show a count and "Clear filters" button

3. **Data Rows**: Each change is displayed with:
   - **operation**: Color-coded badge (INSERT/UPDATE/DELETE)
   - **table**: Fully qualified table name (schema.table)
   - **All columns**: From the affected row
   - Alternating row colors for readability
   - Hover highlighting
